<h3>Memahami Konsep Interface dan Implementasinya dalam Pemrograman Java</h3><figure><img alt="" src="https://cdn-images-1.medium.com/max/750/0*yyQctNc5MEx_ABu2.png" /></figure><p>Dalam dunia pemrograman Java, <strong>interface</strong> merupakan salah satu konsep penting yang memungkinkan pengembang untuk mendefinisikan kontrak yang harus diikuti oleh kelas yang mengimplementasikannya. Interface membantu dalam menciptakan kode yang lebih terstruktur dan mudah dikelola, serta mendukung prinsip pemrograman berorientasi objek, seperti pewarisan (inheritance) dan polimorfisme (polymorphism). Dalam artikel ini, kita akan membahas cara mengimplementasikan interface, karakteristik metode dalam interface, dan perbedaan antara interface dan kelas abstrak. Kami juga akan memberikan contoh nyata tentang cara membuat dan mengimplementasikan interface dalam aplikasi Java, termasuk menyelesaikan soal terkait sistem pembayaran untuk produk Elektronik dan Makanan.</p><h3>Interface</h3><p>Interface dalam Java adalah suatu blueprint yang mendefinisikan metode-metode yang harus diimplementasikan oleh kelas. Deklarasi interface dilakukan dengan menggunakan kata kunci interface.</p><p>Interface hanya dapat mendeklarasikan metode, tanpa implementasi. Kelas yang mengimplementasikan interface harus menyediakan implementasi untuk semua metode yang didefinisikan dalam interface tersebut.</p><h3>Mengimplementasikan Interface</h3><p>Untuk mengimplementasikan interface, kita menggunakan sintaks berikut:</p><pre><modifier> class <name> [extends <super class>] <br />[implements <interface> [, <interface>]*] { <br />    <declarations>* <br />}</pre><p>Sebelum melanjutkan ke contoh implementasi interface yang lebih kompleks, mari kita lihat contoh implementasi interface dalam konteks kelas Line, yang juga dapat dianggap sebagai contoh nyata penggunaan interface di mana kelas ini akan mengimplementasikan metode dari interface yang relevan, seperti Relation.</p><pre>public class Line implements Relation {<br />    private double x1;<br />    private double x2;<br />    private double y1;<br />    private double y2;<br /><br />    public Line(double x1, double x2, double y1, double y2) {<br />        this.x1 = x1;<br />        this.x2 = x2;<br />        this.y1 = y1;<br />        this.y2 = y2;<br />    }<br /><br />    public double getLength() {<br />        double length = Math.sqrt((x2 - x1) * (x2 - x1) + (y2 - y1) * (y2 - y1));<br />        return length;<br />    }<br />}</pre><h3>Penjelasan Kode</h3><ol><li><strong>Deklarasi Kelas</strong>: Kelas Line mengimplementasikan interface Relation, yang berarti kelas ini harus menyediakan implementasi untuk semua metode yang didefinisikan dalam Relation.</li><li><strong>Variabel Anggota</strong>: Kelas ini memiliki empat variabel anggota (x1, x2, y1, y2) yang merepresentasikan koordinat titik-titik di dalam sistem koordinat Cartesian.</li><li><strong>Konstruktor</strong>: Konstruktor Line digunakan untuk menginisialisasi nilai dari variabel anggota ketika objek Line dibuat.</li><li><strong>Metode </strong><strong>getLength()</strong>: Metode ini menghitung panjang garis menggunakan rumus jarak Euclidean antara dua titik dalam ruang dua dimensi. Hasilnya dikembalikan sebagai nilai double.</li></ol><h3>Inheritance pada Interface</h3><p>Interface juga dapat mewarisi dari interface lainnya. Misalnya, kita dapat mendefinisikan interface PersonInterface dan membuat interface StudentInterface yang mewarisi dari PersonInterface:</p><pre>public interface PersonInterface {<br />    void doSomething();<br />}<br />public interface StudentInterface extends PersonInterface {<br />    void doExtraSomething();<br />}</pre><h3>Implementasi Multiple Interface</h3><p>Salah satu keunggulan dari interface adalah kemampuan untuk mengimplementasikan lebih dari satu interface dalam satu kelas. Ini memungkinkan fleksibilitas yang lebih besar dalam desain aplikasi.</p><h3>Kegunaan Interface</h3><p>Interface memiliki banyak kegunaan, di antaranya:</p><ul><li>Menyediakan kontrak yang harus diikuti oleh kelas yang mengimplementasikannya.</li><li>Memungkinkan pengembangan perangkat lunak yang lebih terstruktur.</li><li>Meningkatkan kemampuan untuk mengubah implementasi tanpa mengubah kode yang bergantung pada interface tersebut.</li></ul><h3>Karakteristik Metode dalam Interface</h3><p>Metode dalam interface secara otomatis bersifat public dan abstract, sehingga tidak perlu mendeklarasikannya. Selain itu, dengan diperkenalkannya Java 8, interface dapat memiliki metode default dan static.</p><h3>Perbedaan Interface dan Kelas Abstrak</h3><p>Berikut adalah tabel yang menggambarkan perbedaan utama antara interface dan kelas abstrak:</p><a href="https://medium.com/media/7bf186e89e0e14898aa7e88309687170/href">https://medium.com/media/7bf186e89e0e14898aa7e88309687170/href</a><h3>Persiapan</h3><p>Untuk memulai, ikuti langkah-langkah berikut:</p><ul><li>Buka NetBeans yang sudah terinstal di komputer Anda.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*t4PKPugGFN-CWE0uRjHvOA.png" /></figure><ul><li>Buat proyek baru dengan nama PraktikumPBO_10.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*5ZYIouCRDTCT2DsqdxEgIA.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*EzFME9Tz7I8gfPTStI36WA.png" /></figure><ul><li>Buat package baru dengan nama praktikum10.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/826/1*OUUvmXSlKIJlE-MyfBVFFA.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*ae8HSppvXUZ1uFSNOBs-5g.png" /></figure><h3>Membuat dan Mengimplementasikan Interface</h3><ul><li><strong>Membuat Interface</strong></li></ul><p>Pertama, kita perlu membuat interface yang bernama OperasiHitung. Interface ini akan mendefinisikan satu metode yang disebut hitung, yang menerima dua parameter bertipe int dan mengembalikan nilai bertipe int.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*Ahym7xUDa1ht3l76kJcuDA.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*siZ6s1bAhcU4DvA8WDPvrA.png" /></figure><p><strong>OperasiHitung.java</strong></p><pre>// Interface<br />public interface OperasiHitung {<br />    int hitung(int a, int b);<br />}</pre><p>Mendefinisikan metode hitung(int a, int b) yang harus diimplementasikan oleh kelas yang mengimplementasikan interface ini.</p><ul><li><strong>Membuat Kelas Penjumlahan</strong></li></ul><p>Selanjutnya, kita akan membuat kelas Penjumlahan yang mengimplementasikan interface OperasiHitung. Kelas ini akan menyediakan implementasi untuk metode hitung() yang melakukan penjumlahan.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*mBRZAWsZ0kheXVKCVnqbqQ.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*MZ8_65LR9GcMNOKBLNya5w.png" /></figure><p><strong>Penjumlahan.java</strong></p><pre>// Kelas Penjumlahan<br />public class Penjumlahan implements OperasiHitung {<br />    @Override<br />    public int hitung(int a, int b) {<br />        return a + b;<br />    }<br />}</pre><p>Mengimplementasikan metode hitung() untuk menjumlahkan dua angka.</p><ul><li><strong>Membuat Kelas Pengurangan</strong></li></ul><p>Selain kelas Penjumlahan, kita juga akan membuat kelas Pengurangan yang juga mengimplementasikan interface OperasiHitung. Kelas ini akan melakukan operasi pengurangan.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*9S-pGbD3m8kcVQmEmJ-vXA.png" /></figure><p><strong>Pengurangan.java</strong></p><pre>// Kelas Pengurangan<br />public class Pengurangan implements OperasiHitung {<br />    @Override<br />    public int hitung(int a, int b) {<br />        return a - b;<br />    }<br />}</pre><p>Mengimplementasikan metode hitung() untuk mengurangi dua angka.</p><ul><li><strong>Membuat Kelas Main</strong></li></ul><p>Terakhir, kita akan membuat kelas Main untuk menguji implementasi yang telah kita buat. Dalam kelas ini, kita akan membuat objek dari kelas Penjumlahan dan Pengurangan, kemudian memanggil metode hitung() untuk masing-masing objek dan menampilkan hasilnya.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*gTtPxRrR_iwAMv1qJOlL_g.png" /></figure><p><strong>Main.java</strong></p><pre>// Kelas Main<br />public class Main {<br />    public static void main(String[] args) {<br />        // Objek Penjumlahan<br />        OperasiHitung penjumlahan = new Penjumlahan();<br />        System.out.println("Penjumlahan: " + penjumlahan.hitung(10, 5));<br />        // Objek Pengurangan<br />        OperasiHitung pengurangan = new Pengurangan();<br />        System.out.println("Pengurangan: " + pengurangan.hitung(10, 5));<br />    }<br />}</pre><p>Membuat objek dari Penjumlahan dan Pengurangan, kemudian memanggil metode hitung() untuk menampilkan hasil operasi.</p><ul><li><strong>Jalankan Program</strong></li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*z_VFCN5zZ6xoz2FGUfHLKQ.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*kI800QpDIvHmEaOAw19DWg.png" /></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*ocVfgC-EPcaz-YQ27dVuIA.png" /></figure><p><strong>Output:</strong></p><figure><img alt="" src="https://cdn-images-1.medium.com/max/827/1*qO0q1ns3ro_0_fh48E-G4g.png" /></figure><h3>Soal</h3><p>Anda diminta untuk membuat program sistem pembayaran untuk sebuah toko yang menjual dua jenis produk: Elektronik dan Makanan. Melalui contoh yang diberikan, kita dapat melihat bagaimana interface membantu dalam mendefinisikan metode yang harus diikuti oleh kelas lain, serta cara mereka berkontribusi dalam mendukung prinsip-prinsip desain perangkat lunak yang baik. Program sistem pembayaran yang telah kita buat menunjukkan penerapan praktis dari konsep interface, di mana kita dapat dengan mudah menghitung pajak berdasarkan jenis produk tanpa mengubah struktur dasar kode.
